## Why is Breaking Down Functions Beneficial?

- Improves readability and makes the code easier to understand.
- Simplifies debugging and testing by isolating logic.
- Increases reusability by allowing smaller functions to be used elsewhere.

## How Did Refactoring Improve the Structure of the Code?

- Separated concerns by breaking the function into smaller, focused functions.
- Reduced redundancy by centralizing repeated calculations.
- Made the code modular and easier to maintain.
